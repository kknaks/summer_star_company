# Common Convention

언어 무관, 모든 컴포넌트(backend/agent/admin) 공통.

## 커밋 메시지

형식: `<type>: <한글 요약>`

| type | 용도 |
|---|---|
| `feat` | 새 기능 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 구조 개선 |
| `docs` | 문서만 수정 |
| `chore` | 의존성/설정/잡일 |
| `test` | 테스트만 추가/수정 |

규칙:
- 한 커밋 = 한 논리적 변경 (섞지 말 것)
- 본문은 **왜** 했는지 설명 (무엇은 diff가 보여줌)
- 주제 앞 컴포넌트 prefix 가능: `feat(be):`, `feat(fe):`, `feat(agent):`

예:
```
feat(be): /api/cards/scan 엔드포인트 추가

등록 리더(#2) pull 방식. timeout 30초.
```

## 주석

- 기본은 **주석 없음** — 잘 쓴 이름이 WHAT을 설명함
- 다음 경우에만 주석 작성:
  - 비자명한 WHY (특정 버그 우회, 외부 제약 등)
  - 미묘한 invariant
  - "이거 왜 이렇게 했지?" 가 미래에 일어날 만한 곳
- 다음은 절대 적지 말 것:
  - WHAT 설명 (`# 사용자 조회` 같은 거)
  - PR/이슈 참조 (커밋 메시지에)
  - "added for X feature" 같은 임시 맥락

## 파일 / 모듈

- 파일 헤더 주석 X
- 한 파일 한 책임 (라우터, 모델, 서비스 등 분리)
- 임포트는 표준 라이브러리 → 서드파티 → 로컬 순서, 각 그룹 사이 빈 줄

## SSOT 인용

- 코드 안에 도메인 규칙 주석으로 복사 X (옵시디언 위키 결정과 어긋남)
- 필요하면 docstring/JSDoc에 **링크만** (예: `# 규칙: docs/domain/card.md#등록-흐름`)
- 매직 넘버는 상수로, 그 상수 옆에 SSOT 링크

## 이모지 / 톤

- 코드/주석/커밋에 이모지 X (사용자 명시 요청 시 예외)
- 한국어 OK, 영어 OK, 섞어도 OK. 일관성만

## 환경변수

- 모든 시크릿/설정은 `.env` (gitignore)
- `.env.example`을 항상 함께 갱신 ([[../architecture/security#시크릿-인벤토리]])
- 코드 상수 X — 환경변수로

## 포트 컨벤션

로컬 개발용 포트는 **기본 포트 앞에 `4`를 붙임** — 다른 컨테이너/서비스와의 충돌 회피.

| 서비스 | 기본 | 로컬 |
|---|---|---|
| Postgres | 5432 | **45432** |
| Backend (FastAPI) | 8000 | **48000** |
| Admin Web (Next.js) | 3000 | **43000** |

운영(집 서버)에서는 Nginx 리버스 프록시가 표준 포트(443) 받아서 내부에서 사용.

## 테스트

- 테스트는 결정 사항 검증용으로만 (오버엔지니어링 X)
- 1인 운영 스코프라 단위테스트 강제 X. 통합 동작 확인이 우선
- 추가하면 각 컴포넌트의 `tests/` 디렉토리

# Code Checklist Skill

작업이 끝났다고 선언하기 전에 점검할 항목들.

**작업 끝낼 때 돌려보는 체크리스트** — 항목 미흡하면 마무리하기 전에 보완.

- [ ] 적용된 컨벤션 따랐나? (`code-convention` 참조)
- [ ] 레이어 규칙 위반 없나? (router가 DB 직접 만지지 않았나 등)
- [ ] 타입힌트/타입 누락 없나
- [ ] 매직 넘버/하드코드 시크릿 없나
- [ ] 불필요한 주석/dead code 없나
- [ ] 테스트 깨뜨리지 않았나 (있다면)
- [ ] 환경변수 새로 추가했으면 `.env.example` 갱신했나
- [ ] 도메인 결정 변경했으면 owner 문서 갱신했나 (SSOT)
- [ ] `docs/MAP.md`에 새 문서 등록했나 (있다면)

쓰면서 부족하다 싶으면 그때 항목 추가/수정.
